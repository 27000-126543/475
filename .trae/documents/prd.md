## 1. 产品概述

大型古籍数字化与校勘管理平台是一个面向古籍保护与数字化领域的专业协作系统，支持数字化工程师、校勘专家、审校员和管理员四种角色高效协同完成古籍从影像扫描到电子成书的完整流程。平台通过OCR智能识别、版本差异比对、自动预警和绩效统计等核心能力，解决古籍数字化过程中质量难控、进度难追踪、协同效率低的问题，为文化遗产的数字化保护提供专业工具支撑。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 数字化工程师 | 管理员创建账号 | 上传古籍扫描影像、查看OCR识别结果、人工补录低置信度页面 |
| 校勘专家 | 管理员创建账号 | 接收校勘任务、逐页比对影像与识别文本、标记差异并提交校勘记录 |
| 审校员 | 管理员创建账号 | 复核校勘记录、通过锁定版本或退回校勘专家、注明退回原因 |
| 管理员 | 系统预设 | 管理用户账号、设置质量标准和校勘周期、监控任务进度、查看绩效报表 |

### 2.2 功能模块

1. **工作台首页**：角色驱动的任务看板、待办事项、实时消息通知、数据概览
2. **古籍影像管理**：影像上传与预览、OCR识别与初稿生成、置信度标记与人工补录
3. **校勘工作台**：逐页比对视图（影像+文本并排）、差异标记、校勘记录提交、版本差异报告
4. **审校复核**：校勘记录复核、通过锁定版本与编号生成、退回注明原因
5. **电子古籍库**：已锁定版本浏览、带编号电子古籍下载、版本历史追溯
6. **管理与监控**：质量标准设置、校勘周期管理、任务进度实时监控、超时预警
7. **绩效统计**：专家完成页数、通过率、平均处理时长、绩效对比报表
8. **消息中心**：实时推送通知、消息分类筛选、详情查看与凭证下载

## 3. 核心流程

### 3.1 古籍数字化与校勘主流程

数字化工程师上传古籍扫描影像 → 系统自动OCR识别并生成初稿，记录每页置信度 → 低于阈值的页面标记为"待人工补录" → 工程师补录完成后提交 → 校勘专家接收任务 → 逐页比对影像与识别文本 → 标记差异并上传校勘记录 → 系统自动生成版本差异报告 → 审校员复核校勘记录 → 通过则锁定版本并生成带编号电子古籍 → 不通过则退回校勘专家并注明原因。

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：以墨色（#1a1a2e）和宣纸色（#f5f0e8）为基底，搭配朱砂红（#c0392b）作为强调色，营造古籍书房的沉稳典雅氛围
- **辅助色**：靛青（#2c3e50）用于导航和标题，琥珀色（#d4a574）用于高亮和选中态
- **按钮风格**：圆角矩形（border-radius: 6px），主按钮朱砂红填充，次按钮宣纸色边框，悬停时微上浮带阴影
- **字体**：标题使用 Noto Serif SC（衬线体），正文使用 Noto Sans SC（无衬线体），代码/数据使用 Source Han Mono
- **布局风格**：左侧固定导航栏 + 顶部面包屑 + 主内容区，卡片式布局，信息密度适中
- **图标风格**：线性图标（Lucide风格），统一2px线宽，配合细微的毛笔笔触装饰元素
- **装饰元素**：页面四角使用传统回纹边框装饰，分割线采用祥云纹样

### 4.3 响应式设计

- 桌面端优先设计，主要目标分辨率为1920×1080及以上
- 校勘比对视图等核心功能需至少1280px宽度方可正常使用
- 移动端适配消息通知查看和基础审批操作，复杂操作（如校勘比对）仅桌面端可用
- 触摸优化：平板端支持双指缩放影像、长按标注差异

### 4.4 动效设计

- 页面切换：淡入淡出过渡（200ms）
- 卡片悬停：微上浮（2px）+阴影增强
- 新消息到达：铃铛图标摇摆动画+角标弹入
- 超时预警卡片：边框红色脉冲闪烁
- 进度条：平滑过渡动画（300ms ease-out）
- 数据更新：数字滚动动画（countUp效果）
